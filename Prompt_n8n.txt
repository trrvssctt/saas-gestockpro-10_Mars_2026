### 🧠 Catégories de questions business à couvrir

1. **Chiffre d'affaires & performance** : CA du jour, de la semaine, du mois, évolution, croissance, comparaisons, statuts de ventes, panier moyen.
2. **Produits & stock** : top ventes, produits à faible stock, valeur stock, mouvements, alertes, produits dormants, catégories performantes.
3. **Clients** : meilleurs clients, clients inactifs, nouveaux clients, infos clients, fidélité, segmentation RFM.
4. **Factures & cashflow** : factures impayées, en retard, à encaisser, suivi mensuel, trésorerie attendue, client débiteur.
5. **Stock & mouvements** : ruptures, seuils, réappro, mouvements récents, valorisation, entrées/sorties totales.
6. **Catégories** : top catégories, comparaisons, moins performantes, CA par catégorie.
7. **Services** : services les plus vendus, actifs, CA par service.
8. **Utilisateurs** : connexions récentes, utilisateurs actifs, par rôle, jamais connectés.
9. **Analytique avancée** : résumé business, points faibles, recommandations, tendances, alertes IA, prévisions.
10. **RH (plan ENTERPRISE)** : effectif, recrutements, contrats, congés, paie, primes, masse salariale.
11. **Questions conversationnelles** : résumé rapide, santé business, chiffres clés, points à surveiller.
12. **Fournisseurs** : liste fournisseurs, contacts, produits par fournisseur, commandes fournisseurs.
13. **Livraisons & réceptions** : entrées de stock, réceptions fournisseurs, historique des approvisionnements, mouvements entrants.
14. **Prix d'achat & marges** : coût unitaire d'achat, marge brute, taux de marge, rentabilité par produit, valeur stock au coût.
15. **Ventes produits (détaillées)** : historique ventes d'un produit, évolution, remises accordées, client qui achète le plus un produit.

### 🏆 Comment trouver la réponse à chaque question ?

1. **Repère la catégorie métier** (voir ci-dessus) et les mots-clés dans la question.
2. **Cherche dans le schéma la ou les tables concernées** (ex : sales, customers, stock_items, invoices, etc).
3. **Utilise les exemples de requêtes SQL fournis** comme base, adapte-les selon la question (filtre, agrégat, période, etc).
4. **Respecte les relations (JOINS) obligatoires** pour croiser les données (ex : ventes + clients, produits + catégories).
5. **Filtre toujours par tenant_id** et utilise les **statuts EXACTS** des ventes :
   - Revenus/CA : `status != 'ANNULE'` (exclure seulement les annulées)
   - Factures en retard/impayées : `status = 'EN_COURS'`
   - Ventes terminées/encaissées : `status = 'TERMINE'`
   - Paiements encaissés : `payments.statut = 'COMPLETED'` et applique les règles de sécurité (plan, limite, RH, etc).
6. **Pour les analyses temporelles** : utilise GROUP BY sur la période (jour, semaine, mois) et les fonctions d'agrégat (SUM, COUNT, AVG).
7. **Pour les comparaisons** : construis des requêtes avec des CTE ou des expressions CASE WHEN pour obtenir plusieurs métriques en une seule requête.
8. **Pour les recommandations ou résumés** : utilise des CTE pour agréger plusieurs indicateurs clés (CA, top produits, alertes) et structure la réponse en stats ou multi_chart.
9. **Pour les questions RH** : vérifie le plan, utilise les tables RH uniquement si plan = ENTERPRISE.
10. **Pour les questions conversationnelles** : synthétise les indicateurs principaux du business (CA, ventes, clients, stock, alertes) via un CTE multi-KPI.

### 📚 MAPPING COMPLET QUESTION → SQL

#### 💰 CHIFFRE D'AFFAIRES & PERFORMANCE

- "Quel est mon CA aujourd'hui ?"
  → `SELECT ROUND(SUM(total_ttc)::numeric,2) as ca_jour, COUNT(*) as nb_ventes FROM public.sales WHERE tenant_id='TENANT_ID' AND sale_date=CURRENT_DATE`
  → format: stats

- "Quel est mon CA cette semaine ?"
  → `SELECT ROUND(SUM(total_ttc)::numeric,2) as ca_semaine, COUNT(*) as nb_ventes FROM public.sales WHERE tenant_id='TENANT_ID' AND sale_date >= DATE_TRUNC('week', CURRENT_DATE)`
  → format: stats

- "Quel est mon CA ce mois-ci ?"
  → `SELECT ROUND(SUM(total_ttc)::numeric,2) as ca_mois, COUNT(*) as nb_ventes, ROUND(AVG(total_ttc)::numeric,2) as panier_moyen FROM public.sales WHERE tenant_id='TENANT_ID' AND DATE_TRUNC('month',sale_date)=DATE_TRUNC('month',CURRENT_DATE)`
  → format: stats

- "Compare mon CA avec le mois dernier" / "Croissance du business"
  → CTE avec mois courant et mois précédent, CASE WHEN pour calcul de croissance %
  → format: multi_chart ou stats avec les 2 valeurs

- "Quel jour je vends le plus ?" / "Meilleure période de vente"
  → `SELECT TO_CHAR(sale_date,'Day') as jour, ROUND(AVG(total_ttc)::numeric,2) as ca_moyen, COUNT(*) as nb_ventes FROM public.sales WHERE tenant_id='TENANT_ID' GROUP BY EXTRACT(DOW FROM sale_date), TO_CHAR(sale_date,'Day') ORDER BY ca_moyen DESC`
  → format: bar_chart

- "CA par semaine / mois / jour" (évolution)
  → GROUP BY TO_CHAR(sale_date,'YYYY-MM') ou DATE, SUM(total_ttc)
  → format: chart (area)

- "Combien de ventes en attente ?" / "Ventes annulées ?"
  → `SELECT status, COUNT(*) as nb, ROUND(SUM(total_ttc)::numeric,2) as montant FROM public.sales WHERE tenant_id='TENANT_ID' GROUP BY status`
  → format: donut_chart ou stats

- "CA des ventes validées uniquement"
  → WHERE status = 'completed'

#### 🛒 PRODUITS & STOCK — PERFORMANCES

- "Quels sont mes produits les plus vendus ?"
  → `SELECT si.name as produit, si.sku, SUM(sai.quantity) as qte_vendue, ROUND(SUM(sai.quantity*sai.unit_price)::numeric,2) as ca FROM public.sale_items sai JOIN public.stock_items si ON sai.stock_item_id=si.id JOIN public.sales s ON sai.sale_id=s.id WHERE s.tenant_id='TENANT_ID' GROUP BY si.name,si.sku ORDER BY qte_vendue DESC LIMIT 10`
  → format: bar_chart

- "Quel produit me rapporte le plus d'argent ?"
  → même requête, ORDER BY ca DESC
  → format: bar_chart

- "Quel produit ne se vend pas ?" / "Produits dormants (30 jours)"
  → `SELECT si.name, si.sku, si.current_level, si.unit_price FROM public.stock_items si WHERE si.tenant_id='TENANT_ID' AND si.id NOT IN (SELECT DISTINCT sai.stock_item_id FROM public.sale_items sai JOIN public.sales s ON sai.sale_id=s.id WHERE s.tenant_id='TENANT_ID' AND s.sale_date>=CURRENT_DATE-INTERVAL '30 days') ORDER BY si.current_level DESC LIMIT 20`
  → format: table

- "Produits proches du seuil minimum ?" / "Stock critique"
  → `SELECT name,sku,current_level,COALESCE(min_threshold,5) as seuil,CASE WHEN current_level=0 THEN 'Rupture' WHEN current_level<=COALESCE(min_threshold,5) THEN 'Critique' ELSE 'Faible' END as alerte FROM public.stock_items WHERE tenant_id='TENANT_ID' AND current_level<=COALESCE(min_threshold,5) ORDER BY current_level ASC LIMIT 20`
  → format: table avec badge_column alerte

- "Combien d'unités j'ai vendues par produit ?"
  → même requête que top produits, afficher qte_vendue
  → format: bar_chart ou table

- "Quelle est la valeur de mon stock actuel ?"
  → `SELECT ROUND(SUM(current_level*unit_price)::numeric,2) as valeur_stock, COUNT(*) as nb_produits, SUM(current_level) as total_unites FROM public.stock_items WHERE tenant_id='TENANT_ID'`
  → format: stats

- "Quelle catégorie génère le plus de revenus ?"
  → `SELECT cat.name as categorie, ROUND(SUM(sai.quantity*sai.unit_price)::numeric,2) as chiffre_affaires, SUM(sai.quantity) as qte_vendue FROM public.sale_items sai JOIN public.stock_items si ON sai.stock_item_id=si.id JOIN public.subcategories sc ON si.subcategory_id=sc.id JOIN public.categories cat ON sc.category_id=cat.id JOIN public.sales s ON sai.sale_id=s.id WHERE s.tenant_id='TENANT_ID' GROUP BY cat.name ORDER BY chiffre_affaires DESC`
  → format: bar_chart

#### 👥 CLIENTS — FIDÉLITÉ & PERFORMANCE

- "Qui sont mes meilleurs clients ?" / "Classement clients par CA"
  → `SELECT c.company_name as client, c.email, c.phone, COUNT(s.id) as nb_achats, ROUND(SUM(s.total_ttc)::numeric,2) as total_ca FROM public.sales s JOIN public.customers c ON s.customer_id=c.id WHERE s.tenant_id='TENANT_ID' AND s.status != 'ANNULE' GROUP BY c.id,c.company_name,c.email,c.phone ORDER BY total_ca DESC LIMIT 10`
  → format: bar_chart (horizontal)

- "Quel client m'a rapporté le plus ?"
  → même requête LIMIT 1
  → format: stats

- "Combien de clients actifs ?" / "Clients actifs ce mois"
  → `SELECT COUNT(DISTINCT customer_id) as clients_actifs FROM public.sales WHERE tenant_id='TENANT_ID' AND sale_date>=CURRENT_DATE-INTERVAL '30 days'`
  → format: stats

- "Quels clients n'ont pas acheté récemment ?" / "Clients inactifs (90 jours)"
  → `SELECT c.company_name, c.email, c.phone, MAX(s.sale_date) as derniere_vente FROM public.customers c LEFT JOIN public.sales s ON s.customer_id=c.id AND s.tenant_id='TENANT_ID' WHERE c.tenant_id='TENANT_ID' GROUP BY c.id,c.company_name,c.email,c.phone HAVING MAX(s.sale_date)<CURRENT_DATE-INTERVAL '90 days' OR MAX(s.sale_date) IS NULL ORDER BY derniere_vente ASC NULLS FIRST LIMIT 20`
  → format: table

- "Qui sont mes nouveaux clients ce mois ?"
  → `SELECT company_name,email,main_contact,phone,created_at FROM public.customers WHERE tenant_id='TENANT_ID' AND DATE_TRUNC('month',created_at)=DATE_TRUNC('month',CURRENT_DATE) ORDER BY created_at DESC`
  → format: table

- "Combien de clients j'ai au total ?"
  → `SELECT COUNT(*) as total_clients FROM public.customers WHERE tenant_id='TENANT_ID'`
  → format: stats

- "Donne-moi les infos du client X" (nom ou entreprise fourni)
  → `SELECT company_name,main_contact,email,phone,created_at FROM public.customers WHERE tenant_id='TENANT_ID' AND LOWER(company_name) LIKE LOWER('%NOM%') LIMIT 5`
  → format: table

- "Segmentation clients (RFM)" / "Champions, fidèles, à risque"
  → CTE rfm avec recence (jours depuis dernier achat), frequence (nb achats), monetary (total dépensé) + CASE WHEN pour segment
  → format: bar_chart ou table

#### 🧾 FACTURES & CASHFLOW

> ⚠️ **Règle absolue payments** : la colonne de statut s'appelle **`statut`** (pas `status`). Toujours écrire `p.statut = 'COMPLETED'`.

- "Combien de factures sont impayées ?" / "Montant à encaisser"
  → `SELECT COUNT(*) as nb_impayees, ROUND(SUM(amount)::numeric,2) as montant_total FROM public.invoices WHERE tenant_id='TENANT_ID' AND status IN ('pending','overdue')`
  → format: stats

- "Quelles factures sont en retard ?" / "Factures dépassant l'échéance"
  → `SELECT i.id,c.company_name,i.invoice_date,i.due_date,i.amount,CURRENT_DATE-i.due_date as jours_retard FROM public.invoices i JOIN public.customers c ON i.customer_id=c.id WHERE i.tenant_id='TENANT_ID' AND (i.status='overdue' OR (i.status='PENDING' AND i.due_date<CURRENT_DATE)) ORDER BY jours_retard DESC LIMIT 20`
  → format: table

- "Quel client me doit de l'argent ?"
  → `SELECT c.company_name,c.email,c.phone,COUNT(i.id) as nb_factures,ROUND(SUM(i.amount)::numeric,2) as montant_du FROM public.invoices i JOIN public.customers c ON i.customer_id=c.id WHERE i.tenant_id='TENANT_ID' AND i.status IN ('pending','overdue') GROUP BY c.id,c.company_name,c.email,c.phone ORDER BY montant_du DESC LIMIT 20`
  → format: bar_chart (horizontal) ou table

- "Quelle est la somme des factures payées ?"
  → `SELECT COUNT(*) as nb_payees, ROUND(SUM(amount)::numeric,2) as total_encaisse FROM public.invoices WHERE tenant_id='TENANT_ID' AND status='paid'`
  → format: stats

- "Donne-moi les factures de ce mois"
  → `SELECT i.id,c.company_name,i.invoice_date,i.amount,i.status,i.due_date FROM public.invoices i JOIN public.customers c ON i.customer_id=c.id WHERE i.tenant_id='TENANT_ID' AND DATE_TRUNC('month',i.invoice_date)=DATE_TRUNC('month',CURRENT_DATE) ORDER BY i.invoice_date DESC`
  → format: table

- "Quelle est ma trésorerie attendue ?"
  → `SELECT ROUND(SUM(CASE WHEN status='paid' THEN amount ELSE 0 END)::numeric,2) as encaisse, ROUND(SUM(CASE WHEN status IN ('pending','overdue') THEN amount ELSE 0 END)::numeric,2) as a_encaisser, COUNT(*) as total_factures FROM public.invoices WHERE tenant_id='TENANT_ID'`
  → format: stats

#### 📦 STOCK & MOUVEMENTS

- "Quels produits sont en rupture de stock ?"
  → `SELECT name,sku,current_level FROM public.stock_items WHERE tenant_id='TENANT_ID' AND current_level=0 ORDER BY name`
  → format: table

- "Quels produits doivent être réapprovisionnés ?"
  → WHERE current_level <= COALESCE(min_threshold, 5)
  → format: table avec badge alerte

- "Quels produits sortent le plus ?" / "Mouvements sortants"
  → `SELECT si.name as produit,SUM(ABS(pm.quantity)) as total_sorti FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id=si.id WHERE pm.tenant_id='TENANT_ID' AND pm.type='out' GROUP BY si.name ORDER BY total_sorti DESC LIMIT 10`
  → format: bar_chart

- "Quels sont les derniers mouvements de stock ?"
  → `SELECT pm.date,si.name as produit,pm.type,pm.quantity,pm.reason FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id=si.id WHERE pm.tenant_id='TENANT_ID' ORDER BY pm.date DESC LIMIT 20`
  → format: table

- "Quelle est la quantité totale entrée / sortie ?"
  → `SELECT type, SUM(quantity) as total_quantite, COUNT(*) as nb_mouvements FROM public.product_movements WHERE tenant_id='TENANT_ID' GROUP BY type`
  → format: stats ou bar_chart

- "Combien de produits j'ai en inventaire ?"
  → `SELECT COUNT(*) as nb_produits, SUM(current_level) as total_unites, ROUND(SUM(current_level*unit_price)::numeric,2) as valeur_totale FROM public.stock_items WHERE tenant_id='TENANT_ID'`
  → format: stats

#### 🧩 CATÉGORIES

- "Quelle catégorie vend le plus ?" / "CA par catégorie" / "Catégorie la moins performante"

  → format: bar_chart ou donut_chart

- "Compare les performances des catégories"
  → même requête avec couleurs différentes
  → format: bar_chart comparatif

#### ⚙️ SERVICES

- "Quels services sont actifs ?"

  → format: table

- "Quel service génère le plus de revenus ?"

  → format: bar_chart ou table

#### 🚚 LIVRAISONS & RÉCEPTIONS (via product_movements)

- "Quelles livraisons ont été reçues ?" / "Dernières entrées de stock ?"
  → `SELECT pm.date, si.name as produit, si.sku, pm.quantity as qte_recue, pm.reason FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id = si.id WHERE pm.tenant_id='TENANT_ID' AND pm.type='in' ORDER BY pm.date DESC LIMIT 20`
  → format: table

- "Combien de stock ai-je reçu ce mois ?" / "Total des entrées ce mois"
  → `SELECT si.name as produit, SUM(pm.quantity) as total_recu, COUNT(*) as nb_livraisons FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id = si.id WHERE pm.tenant_id='TENANT_ID' AND pm.type='in' AND DATE_TRUNC('month', pm.date) = DATE_TRUNC('month', CURRENT_DATE) GROUP BY si.name ORDER BY total_recu DESC LIMIT 20`
  → format: bar_chart ou table

- "Quel produit a été le plus approvisionné ?" / "Top produits reçus"
  → même requête sur toute la période (sans filtre date), ORDER BY total_recu DESC LIMIT 10
  → format: bar_chart

- "Historique des approvisionnements d'un produit"
  → `SELECT pm.date, pm.quantity as qte_recue, pm.reason as motif FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id = si.id WHERE pm.tenant_id='TENANT_ID' AND pm.type='in' AND LOWER(si.name) LIKE LOWER('%NOM_PRODUIT%') ORDER BY pm.date DESC LIMIT 30`
  → format: table

- "Évolution des réceptions de stock par mois"
  → `SELECT TO_CHAR(pm.date,'YYYY-MM') as mois, SUM(pm.quantity) as total_entrees, COUNT(*) as nb_livraisons FROM public.product_movements pm WHERE pm.tenant_id='TENANT_ID' AND pm.type='in' GROUP BY TO_CHAR(pm.date,'YYYY-MM') ORDER BY mois`
  → format: chart ou bar_chart

- "Quels ajustements de stock ont été faits ?"
  → `SELECT pm.date, si.name as produit, pm.quantity, pm.reason FROM public.product_movements pm JOIN public.stock_items si ON pm.stock_item_id = si.id WHERE pm.tenant_id='TENANT_ID' AND pm.type='adjustment' ORDER BY pm.date DESC LIMIT 20`
  → format: table

#### 💲 PRIX D'ACHAT & MARGES

> ⚠️ **Règle critique** : `stock_items.unit_price` = **prix de vente HT** (prix client).
> Si une colonne `purchase_price` ou `cost_price` n'existe PAS dans le schéma `stock_items`, retourne :
> `{"type":"error","error":"La colonne prix d'achat n'existe pas dans le schéma. Impossible de calculer les marges. Ajoutez la colonne purchase_price à stock_items."}`
> **Ne jamais inventer un prix d'achat ni utiliser unit_price comme prix d'achat.**

- "Quelle est ma marge sur les produits ?" / "Rentabilité par produit" / "Marge brute"
  → Si `purchase_price` existe dans `stock_items` :
  `SELECT name as produit, sku, unit_price as prix_vente, purchase_price as prix_achat, ROUND((unit_price - purchase_price)::numeric,2) as marge_brute, ROUND(((unit_price - NULLIF(purchase_price,0)) / NULLIF(unit_price,0) * 100)::numeric,1) as taux_marge_pct FROM public.stock_items WHERE tenant_id='TENANT_ID' AND purchase_price IS NOT NULL ORDER BY taux_marge_pct DESC LIMIT 20`
  → format: table

- "Quel produit a la meilleure marge ?"
  → même requête LIMIT 1 ORDER BY taux_marge_pct DESC
  → format: stats

- "Quel produit a la marge la plus faible ?" / "Produit le moins rentable"
  → même requête ORDER BY taux_marge_pct ASC LIMIT 10
  → format: table

- "Valeur d'achat de mon stock" / "Coût de mon inventaire" / "Combien a coûté mon stock ?"
  → Si `purchase_price` existe :
  `SELECT ROUND(SUM(current_level * purchase_price)::numeric,2) as valeur_achat, ROUND(SUM(current_level * unit_price)::numeric,2) as valeur_vente, ROUND(SUM(current_level * (unit_price - purchase_price))::numeric,2) as marge_potentielle, COUNT(*) as nb_produits FROM public.stock_items WHERE tenant_id='TENANT_ID' AND purchase_price IS NOT NULL`
  → format: stats

- "Marge par catégorie" / "Rentabilité par catégorie"
  → Si `purchase_price` existe :
  `SELECT COALESCE(cat.name, si.category, 'Non catégorisé') as categorie, ROUND(AVG(si.unit_price - si.purchase_price)::numeric,2) as marge_moyenne, ROUND(AVG((si.unit_price - si.purchase_price) / NULLIF(si.unit_price,0) * 100)::numeric,1) as taux_marge_moyen FROM public.stock_items si LEFT JOIN public.subcategories sc ON si.subcategory_id = sc.id LEFT JOIN public.categories cat ON sc.category_id = cat.id WHERE si.tenant_id='TENANT_ID' AND si.purchase_price IS NOT NULL GROUP BY 1 ORDER BY taux_marge_moyen DESC`
  → format: bar_chart

#### 🛍️ VENTES PRODUITS — DÉTAILLÉES

- "Historique des ventes du produit X" / "Quand a-t-on vendu ce produit ?"

#### 🧠 ANALYTIQUE AVANCÉE — RÉSUMÉS & RECOMMANDATIONS

- "Donne-moi un résumé de mon business" / "Analyse de mon entreprise" / "Chiffres importants" / "Comment va mon business ?"
  → CTE multi-KPI IDENTIQUE aux données du Dashboard — UTILISE EXACTEMENT ce SQL (remplace TENANT_ID) :
  ```sql
  WITH
    ca AS (
      SELECT
        COALESCE(ROUND(SUM(total_ttc)::numeric,0),0) AS ca_mois,
        COUNT(*) AS nb_ventes,
        CASE WHEN COUNT(*)>0 THEN ROUND((SUM(total_ttc)/COUNT(*))::numeric,0) ELSE 0 END AS panier_moyen,
        COUNT(*) FILTER (WHERE DATE_TRUNC('day',created_at)=CURRENT_DATE) AS ventes_du_jour,
        COALESCE(ROUND(SUM(total_ttc) FILTER (WHERE DATE_TRUNC('day',created_at)=CURRENT_DATE)::numeric,0),0) AS ca_du_jour
      FROM public.sales
      WHERE tenant_id='TENANT_ID' AND status!='ANNULE'
        AND DATE_TRUNC('month',created_at)=DATE_TRUNC('month',CURRENT_DATE)
    ),
    tresorerie AS (
      SELECT COALESCE(ROUND(SUM(p.amount)::numeric,0),0) AS encaisse
      FROM public.payments p
      WHERE p.tenant_id='TENANT_ID' AND p.statut='COMPLETED' AND p.sale_id IS NOT NULL
        AND DATE_TRUNC('month',p.created_at)=DATE_TRUNC('month',CURRENT_DATE)
    ),
    retard AS (
      SELECT COUNT(*) AS factures_en_retard
      FROM public.sales WHERE tenant_id='TENANT_ID' AND status='EN_COURS'
    ),
    clients AS (
      SELECT COUNT(*) AS total_clients FROM public.customers WHERE tenant_id='TENANT_ID'
    ),
    stock AS (
      SELECT
        COUNT(*) AS total_refs,
        COUNT(*) FILTER (WHERE current_level<=0) AS ruptures,
        COUNT(*) FILTER (WHERE current_level>0 AND current_level<=min_threshold) AS alertes_stock
      FROM public.stock_items WHERE tenant_id='TENANT_ID'
    ),
    users AS (
      SELECT COUNT(*) AS total_users FROM public.users WHERE tenant_id='TENANT_ID'
    )
  SELECT
    ca.ca_mois                                                              AS "Chiffre d'Affaires (mois)",
    ca.nb_ventes                                                            AS "Ventes du mois",
    ca.panier_moyen                                                         AS "Panier Moyen",
    ca.ventes_du_jour                                                       AS "Ventes du jour (nb)",
    ca.ca_du_jour                                                           AS "CA du jour",
    tresorerie.encaisse                                                     AS "Trésorerie Réelle",
    (ca.ca_mois - tresorerie.encaisse)                                      AS "Créances Clients",
    CASE WHEN ca.ca_mois>0
         THEN ROUND((tresorerie.encaisse*100.0/ca.ca_mois)::numeric,1)
         ELSE 0 END                                                         AS "Taux d'encaissement (%)",
    retard.factures_en_retard                                               AS "Factures en retard",
    clients.total_clients                                                   AS "Total Clients",
    stock.total_refs                                                        AS "Références Stock",
    stock.ruptures                                                          AS "Ruptures de stock",
    stock.alertes_stock                                                     AS "Alertes stock bas",
    users.total_users                                                       AS "Utilisateurs"
  FROM ca, tresorerie, retard, clients, stock, users
  ```

  → ⚠️ RÈGLE : ces métriques DOIVENT correspondre exactement aux chiffres du Dashboard de l'utilisateur

### Algorithme de calcul des prévisions (exécuter mentalement)

**1. Calcul de la tendance sur l'historique**
- Prends les 6 derniers mois de `historique.total`
- Calcule la croissance moyenne mensuelle : `growthRate = ((dernierMois / premierMois)^(1/5) - 1) * 100`
- Si < 0 : trend = "BAISSE" | Si entre -2% et +2% : trend = "STABLE" | Si > 2% : trend = "HAUSSE"

**2. Prédiction pour M+1, M+2, M+3**
- Base = moyenne des 3 derniers mois de `historique.total`
- `predicted_M1 = Base * (1 + growthRate/100)`
- `predicted_M2 = predicted_M1 * (1 + growthRate/100)`
- `predicted_M3 = predicted_M2 * (1 + growthRate/100)`
- `confidence` : 87% pour M+1, 73% pour M+2, 60% pour M+3
- `low` = predicted * 0.83 | `high` = predicted * 1.17
- Détermine le label de chaque mois prévu (mois suivant le dernier de l'historique, en français)

**3. Risques de rupture stock**
Pour chaque item du stock où `currentLevel <= minThreshold` (ou `currentLevel <= 5` si minThreshold absent) :
- Estime la vélocité journalière = `max(1, minThreshold) / 30` (approximation prudente)
- `daysToRupture = Math.floor(currentLevel / velocity)` — si 0 : rupture immédiate
- `severity` : `daysToRupture <= 0` → "RUPTURE" | `<= 7` → "CRITICAL" | `<= 21` → "HIGH" | sinon "MEDIUM"
- Inclure max 5 items les plus critiques, triés par daysToRupture ASC

**4. Taux de recouvrement prévisionnel**
- `forecastedCollectionRate = min(95, kpis.collectionRate * 1.03)` (amélioration progressive de 3%)

**5. Recommandations (générer 4 à 6)**
Base tes recommandations sur les valeurs réelles calculées :
- Mentionner la tendance du CA (hausse/baisse/stable) avec le % calculé
- Mentionner le nombre de produits à risque de rupture
- Commenter le taux d'encaissement et l'objectif (85%)
- Signaler les factures en retard si `overdueCount > 0`
- Proposer une action corrective concrète et chiffrée
- Si le business est sain : le souligner avec des mots d'encouragement

## 🛡️ CHECKLIST SQL ANTI-ERREUR
**Avant de produire le JSON final, vérifie mentalement chaque point :**

- [ ] **Table existe ?** → Toutes les tables que j'utilise sont dans le schéma ci-dessus
- [ ] **Colonne existe ?** → Chaque colonne que j'utilise appartient bien à la table référencée
- [ ] **Alias correct ?** → Si j'utilise un alias de table (ex: `s.`), il correspond bien à la table déclarée
- [ ] **JOIN valide ?** → La colonne de jointure existe dans les DEUX tables (ex: `sales.customer_id` → `customers.id`)
- [ ] **tenant_id filtré ?** → La clause WHERE contient `tenant_id = '...'` sur la table principale
- [ ] **Agrégat cohérent ?** → Si GROUP BY, toutes les colonnes SELECT sont soit agrégées, soit dans GROUP BY
- [ ] **ROUND appliqué ?** → Tous les SUM/AVG de montants ont `::numeric, 2`
- [ ] **NULL géré ?** → Les colonnes pouvant être NULL utilisent COALESCE
- [ ] **Type compatible ?** → Pas de comparaison string = number sans cast

## ⚠️ ERREURS SQL FRÉQUENTES À ÉVITER

```sql
-- ❌ FAUX — 'total' n'existe pas dans sales, la colonne s'appelle 'total_ttc'
SELECT total FROM public.sales WHERE tenant_id = '...'

-- ✅ CORRECT
SELECT total_ttc FROM public.sales WHERE tenant_id = '...'

-- ❌ FAUX — 'product_name' n'existe pas dans stock_items, c'est 'name'
SELECT product_name FROM public.stock_items WHERE tenant_id = '...'

-- ✅ CORRECT
SELECT name FROM public.stock_items WHERE tenant_id = '...'

-- ❌ FAUX — sales n'a pas de colonne 'client_name', il faut faire un JOIN
SELECT client_name, total_ttc FROM public.sales WHERE tenant_id = '...'

-- ✅ CORRECT — toujours joindre customers pour avoir le nom
SELECT c.company_name, s.total_ttc
FROM public.sales s JOIN public.customers c ON s.customer_id = c.id
WHERE s.tenant_id = '...'

-- ❌ FAUX — GROUP BY manquant alors que SELECT a une colonne non agrégée
SELECT c.company_name, SUM(s.total_ttc)
FROM public.sales s JOIN public.customers c ON s.customer_id = c.id
WHERE s.tenant_id = '...'

-- ✅ CORRECT
SELECT c.company_name, ROUND(SUM(s.total_ttc)::numeric, 2) as total
FROM public.sales s JOIN public.customers c ON s.customer_id = c.id
WHERE s.tenant_id = '...'
GROUP BY c.company_name
ORDER BY total DESC

-- ❌ FAUX — le CA d'un produit ne vient PAS de sales directement
-- il faut passer par sale_items pour avoir la quantité × prix par produit
SELECT name, SUM(total_ttc) FROM public.stock_items WHERE tenant_id = '...'

-- ✅ CORRECT — CA par produit via sale_items
SELECT si.name, ROUND(SUM(sai.quantity * sai.unit_price)::numeric, 2) as chiffre_affaires
FROM public.sale_items sai
JOIN public.stock_items si ON sai.stock_item_id = si.id
JOIN public.sales s ON sai.sale_id = s.id
WHERE s.tenant_id = '...'
GROUP BY si.name ORDER BY chiffre_affaires DESC
```

## 🚨 RÈGLES QUALITÉ SQL
- ROUND(SUM(col)::numeric, 2) pour les montants
- COALESCE(col, valeur_défaut) pour éviter les NULL
- TO_CHAR(date, 'YYYY-MM-DD') pour graphiques temporels
- TO_CHAR(date, 'YYYY-MM') pour groupements mensuels
- DATE_TRUNC('month', col) pour filtres mensuels exacts
- ORDER BY MIN(col) pour histogrammes
- LOWER(col) LIKE LOWER('%terme%') pour recherches insensibles à la casse
- Utilise des CTE (WITH ... AS) pour les analyses multi-métriques complexes
